# Origami-Sonobe-Graph-Coloring

A self-directed project applying graph coloring theory to Sonobe origami, a modular origami form where unit pieces interlock into geometric structures. Each unit is modeled as a vertex in a graph, and valid three-color arrangements are found such that no two adjacent units share the same color. This frames the physical assembly of origami as an instance of the classical graph coloring problem, a well-studied NP-complete problem closely related to the Four Color Theorem.

## Approach

The project follows three stages, each in a different tool:

1. 3D Modeling in Blender. Twelve Sonobe models of varying sizes were built from scratch: 3, 6, 12, 24 (Hexahedron), 30, 36, 48, 54, 90, 108, 120, and 144-piece configurations.
2. Graph Representation in Wolfram. Each 3D model was translated into a planar graph, with each Sonobe unit represented as a vertex and each adjacency as an edge. Wolfram's default GraphPlot layout algorithm produced unusable results on larger models. Vertices near the outer boundary had longer edges that pulled interior vertices inward, causing severe clumping in the center. Even on the 6-piece model, edges began to overlap, obscuring the underlying structure.
To address this, GraphLayout -> "TutteEmbedding" was used. This is a standard algorithm for drawing planar graphs that positions each vertex at the average position of its neighbors. It naturally forces all faces to be convex and guarantees no edge crossings, producing clean, symmetric layouts that reflect the geometric nature of the Sonobe models. However, Tutte Embedding alone wasn't sufficient for the largest models. While edge crossings were eliminated, vertices on models like the 144-piece configuration still clustered too densely to read individual node numbers, requiring the additional visualization techniques discussed later.
3. Coloring in Python. Custom Graph and Node classes were implemented, along with a backtracking algorithm that assigns one of three colors (Red / Green / Blue) to each node such that no two adjacent nodes share a color.

## Algorithm Design

The initial implementation attempted to enumerate all valid colorings through brute force, iterating every node-to-node combination and validating each configuration. This worked for small models (3 and 6 pieces), but broke down on the 12-piece model, where the combinatorial space reached approximately 28¹² ≈ 2.3 × 10¹⁷ configurations.
Rather than optimize the brute-force approach, the strategy was redesigned. Instead of searching for all valid colorings, the new algorithm searches for one valid coloring, mirroring how one might physically assemble the model by prioritizing color placement from the outside in:

1. Layer decomposition: Nodes are grouped into layers based on their structural position, moving from the outermost layer inward.
2. Fixed starting pattern: The first three nodes of the outer layer are hardcoded as R-G-B to reduce the search space.
3. Backtracking with layer awareness: Each subsequent node is assigned a color that doesn't conflict with its neighbors. If a conflict becomes unavoidable, the algorithm backtracks to the most recent decision point and tries the alternative.

This approach produces a single valid coloring for each of the twelve models, including the 144-piece configuration that the brute-force method couldn't handle.

## Visualization Challenges

Rendering large graphs legibly required additional work in Wolfram. On the 144-piece model, vertices clumped so densely that individual node numbers became unreadable. Two techniques were used to address this:

* Coordinate scaling: Positions generated by TutteEmbedding were scaled down (up to 15× on the largest model) to reduce vertex size and spacing.
* Progressive vertex deletion: Once outer-layer positions were confirmed, those vertices were removed from the plot to make inner layers inspectable. This also influenced how layers were defined. Rather than purely by Euclidean distance from center, layers were ordered so that removing one wouldn't leave another disconnected.

## Verification

Each coloring produced by the Python algorithm was pasted back into Wolfram 
and rendered on the original graph layout. This allowed visual confirmation 
that no two adjacent vertices shared the same color across all twelve models, 
closing the loop between algorithmic output and the original geometric structure.
